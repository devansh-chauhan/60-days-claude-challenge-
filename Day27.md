# Day 27 – Prior Authorization Story Simulator

##  Overview

This project was completed as part of **Day 27 of the 60 Days of Claude Challenge**.

The objective was to build an interactive **Prior Authorization Story Simulator** that explains the U.S. healthcare Prior Authorization (PA) process through a beginner-friendly, story-driven experience.

Unlike a traditional workflow dashboard, this application presents the journey as an interactive conversation between **Rahul (Patient)** and **Priya (Healthcare Operations Specialist)**. Users progress through eight chapters, make choices at every stage, and learn how insurance prior authorization works from diagnosis to final approval.

---

##  Project Objectives

* Explain Prior Authorization using storytelling
* Simplify healthcare concepts for beginners
* Demonstrate the patient and provider journey
* Build an interactive educational web application
* Practice dynamic DOM manipulation using Vanilla JavaScript

---

##  Features

###  Interactive Story

* Eight sequential story chapters
* Rahul (Patient) conversation
* Priya (Healthcare Operations Specialist) guidance
* Doctor and narrator messages
* Beginner-friendly healthcare explanations

###  Prior Authorization Journey

* Doctor consultation
* Insurance roadblock
* Understanding Prior Authorization
* Insurance review process
* Denial scenario
* Appeal process
* Approval workflow
* Final learning summary

###  Interactive Experience

* Two decision choices after every chapter
* Dynamic append-only chat interface
* Progress bar tracking story completion
* Multiple dialogue paths
* Restart story functionality

###  Educational Content

* Medical necessity explanation
* Step therapy overview
* Insurance eligibility checks
* Clinical documentation requirements
* Appeal documentation
* Healthcare system metrics
* Patient takeaways

---

##  Technologies Used

* HTML5
* Tailwind CSS
* Vanilla JavaScript

The application is implemented as a **single self-contained HTML file** with no backend or external frameworks beyond Tailwind CSS.
