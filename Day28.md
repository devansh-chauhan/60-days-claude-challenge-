# 🏥 Day 28 – Hospital Admission Readiness Simulator

## Overview

This project is a **Hospital Admission Readiness Simulator** built as part of the **60 Days of Claude Challenge (Day 28)**. It is a single-file HTML application that simulates the workflow followed by hospital admission coordinators when preparing patients for admission.

The simulator focuses on healthcare operations, prior authorization, documentation readiness, insurance verification, bed allocation, and clinical coordination while providing an educational, interactive experience.

## Features

* Provider and physician setup
* Multiple diagnosis scenarios

  * Acute Myocardial Infarction (MI)
  * Congestive Heart Failure (CHF)
  * Pneumonia
  * Elective Surgery
  * Hip Fracture
* Admission type selection

  * Inpatient
  * Observation
  * Emergency
  * ICU
  * Same-Day Surgery
* Prior Authorization workflow

  * Approved
  * Pending
  * Denied
  * Appeal process
* Admission Readiness Score calculation
* Risk tracking dashboard

  * Documentation Risk
  * Insurance Risk
  * Bed Availability Risk
  * Clinical Risk
* Timeline milestones from PA review to admission completion
* Care Coordination cards

  * Attending Physician
  * Nursing
  * Case Management
  * Utilization Review
  * Discharge Planning
* Governance Snapshot with healthcare industry benchmark estimates
* Final admission decision based on readiness score
* Responsive healthcare-themed interface
* Restart simulator with different patient scenarios

## Technologies Used

* HTML5
* Tailwind CSS
* Vanilla JavaScript

## Learning Outcomes

Through this project, I learned:

* Hospital admission readiness workflow
* Prior Authorization decision paths
* Insurance verification processes
* Clinical documentation requirements
* Risk assessment in patient admissions
* Care coordination across hospital departments
* Readiness scoring and workflow automation
* Healthcare simulation UI design using JavaScript
