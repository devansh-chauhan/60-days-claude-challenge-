# Day 9 – NutriScope: MVP vs Enhanced Version

## Challenge Objective

Today's challenge focused on understanding how AI can be used not only to build an application from scratch but also to iteratively improve and scale an existing product.

The task was to build a nutrition tracking application called **NutriScope** in two stages:

1. Build a Minimum Viable Product (MVP)
2. Enhance the MVP with advanced features

# Key Learnings

## 1. AI Can Build Products Iteratively

Instead of creating everything at once, AI can progressively improve a product through multiple prompts.

## 2. MVP First, Scale Later

The MVP focused on solving the core problem.

The enhanced version focused on usability, analytics, and advanced functionality.

## 3. Prompt Chaining Is Powerful

The second prompt did not rebuild the project from scratch.

It expanded the existing product by adding meaningful features.

## 4. Product Development Mirrors Real Software Engineering

The exercise followed a real-world workflow:

Idea → MVP → Feedback → Enhancements → Better Product

---

# Biggest Insight

The most important lesson from Day 9 was learning that AI works best when treated like a development partner.

Instead of asking for a perfect application in one prompt, it is often more effective to:

1. Build an MVP
2. Test the result
3. Improve through iterative prompting

This closely resembles how real startups and software teams build products.
